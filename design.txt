The key pattern here:

1. **Context flows down**: Each level receives context from its parent and adds to it
2. **Results flow up**: Each recursive call returns its portion of the OpenAPI spec
3. **The stack manages state**: `new_context` variables at each level are automatically preserved

Each nested level inherits and extends the context, and as you return up the stack, you merge the generated OpenAPI paths. No AST needed - just function calls and return values!
